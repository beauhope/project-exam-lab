# محاكي إدارة المشاريع الاحترافي

تطبيق ويب ثابت عربي مستقل للتدريب على اختبارات إدارة المشاريع المهنية. يعتمد على بنك أصلي من 500 سؤال، ويعمل دون أطر عمل أو خادم خلفي أو خدمات خارجية. لا يحمل التطبيق شعار جهة مانحة ولا يدّعي أن نتيجته معيار نجاح رسمي.

## التشغيل محليًا

من داخل مجلد المشروع شغّل:

```powershell
py -m http.server 8080
```

ثم افتح:

```text
http://localhost:8080
```

لا يُدعم التشغيل المباشر عبر `file://` لأن تحميل JSON وService Worker يحتاجان إلى HTTP أو HTTPS.

## البنية

- `index.html`: الصفحة الرئيسية، الأوضاع، التدريب المخصص والسجل.
- `exam.html`: شاشة الاختبار.
- `results.html`: تحليل النتيجة والطباعة والتصدير.
- `review.html`: مراجعة الإجابات والتفسيرات.
- `about.html` و`privacy.html`: المعلومات القانونية والخصوصية.
- `assets/css/`: التنسيق العام وتنسيق الطباعة.
- `assets/js/`: وحدات التحميل والاختيار والعرض والحفظ والنتائج.
- `data/questions.json`: المصدر الوحيد لبنك الأسئلة.
- `tools/validate.html`: تقرير تحقق مرئي.
- `tools/validate-data.js`: تحقق آلي من سطر الأوامر.
- `service-worker.js`: التخزين المؤقت والعمل دون اتصال.

## التحقق من البيانات

بعد تشغيل الخادم افتح `http://localhost:8080/tools/validate.html`، أو شغّل:

```powershell
node tools/validate-data.js
```

يتحقق التقرير من العدد والتوزيعات والمعرفات والتفسيرات والإجابات ومراجع دراسات الحالة.

## تحديث بنك الأسئلة

1. استبدل محتوى `data/questions.json` مع الحفاظ على البنية واعتبار `qid` المفتاح الفريد.
2. لا تضع نسخة من البنك داخل HTML أو JavaScript.
3. شغّل أداة التحقق وأصلح الأخطاء الموثقة فقط.
4. حدّث إصدار Cache بعد أي تغيير منشور.

## تحديث Cache

غيّر `CACHE_NAME` في `service-worker.js`، مثلًا من `project-exam-lab-v1` إلى `project-exam-lab-v2`. ويمكن تحديث `cacheVersion` في `assets/js/config.js` للتوثيق. سيحذف Service Worker النسخ القديمة عند التفعيل، بينما تستخدم الصفحات الشبكة أولًا كي لا تمنع ظهور التحديثات.

## النشر عبر GitHub Pages

1. ارفع الملفات إلى مستودع GitHub دون تغيير المسارات النسبية.
2. من إعدادات المستودع افتح **Pages**.
3. اختر النشر من فرع، ثم اختر الفرع الرئيسي والمجلد الجذري `/root`.
4. احفظ وانتظر رابط النشر. وجود `.nojekyll` يمنع معالجة Jekyll غير اللازمة.

جميع روابط التطبيق نسبية، لذلك يعمل من مجلد فرعي مثل `https://user.github.io/repository/`.

## استخدام GitHub Desktop

1. اختر **Add an Existing Repository from your Hard Drive** وحدد مجلد المشروع.
2. راجع الملفات المتغيرة في تبويب Changes.
3. اكتب ملخصًا واضحًا ثم اختر **Commit to main**.
4. اختر **Push origin** لرفع التغييرات.
5. فعّل GitHub Pages من موقع GitHub كما في الخطوات السابقة.

## الخصوصية والقيود

- تُحفظ المحاولة وآخر عشر نتائج محليًا في المتصفح فقط.
- لا توجد حسابات مستخدمين.
- لا تتم مزامنة النتائج بين الأجهزة أو المتصفحات.
- حذف بيانات المتصفح يحذف السجل والمحاولة المحفوظة.
- ملف الأسئلة متاح للعامة ضمن ملفات الموقع الثابت.
- لا توجد قاعدة بيانات أو API خارجي.

## الاختبارات المنفذة

- صلاحية JSON وتحميل 500 سؤال والتوزيعات المعتمدة.
- عدم تكرار `qid` وصحة الإجابات ودراسات الحالة.
- اختيار 180 و60 و30 سؤالًا فريدًا ومتوازنًا.
- أنواع الاختيار المفرد والمتعدد والترتيب والمطابقة.
- الحفظ والاستعادة، المؤقت، التسليم والاستراحات.
- النتائج والسجل ومراجعة الأخطاء والطباعة وتصدير JSON.
- الوضع الداكن، المسارات النسبية، وعدم وجود أخطاء تركيب JavaScript.
- التحميل عبر مجلد فرعي والعمل دون اتصال بعد الزيارة الأولى.
